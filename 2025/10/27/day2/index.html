10步速通-如何在github上贡献开源项目

初始化与环境配置

fork原项目并clone到本地

  • 在项目 (Fork) 在 GitHub 页面上，点击上游仓库 (github.com/remote/test) 右上角的 Fork 按钮，将项目复刻到您个人的账户下（https://www.github.com/me/test）。
  • 将您个人账户下的复刻仓库 (Origin) 克隆到本地开发环境，这会自动创建一个与项目同名的文件夹，并设置好默认的 origin 远程连接。

拉取最新分支并更新

  • 为了能追踪原始项目的更新，需要手动添加一个指向 Upstream 的远程仓库源，我们手动将当前分支切换到main（或master分支）
    ，并合并远程仓库源的最新提交以更新。

在本地创建新分支

  • 切勿直接在 main 分支上进行开发，应基于同步后的 main 分支创建一个新的功能分支。分支命名建议遵循 feat/xxx (新功能) 或 fix/xxx (修复 Bug) 的规范。

将修改的内容暂存、提交、推送到自己的仓库

  • 在当前的功能分支上进行代码编写、修改或测试工作,将修改后的文件添加到 Git 暂存区，提交,最后将您本地完成的功能分支推送到您自己的远程仓库。

在自己的仓库发起Pull request

  • 访问您自己的 GitHub 仓库页面 (github.com/me/test)，点击 “Compare & pull request” 按钮。
    在 PR 页面中：
    确保 base repository 是上游仓库 (remote/test)，base 分支是 main。
    确保 head repository 是您的仓库 (me/test)，compare 分支是您的功能分支 (feat/new-user-module)。
    填写详细的 PR 描述，说明您的改动内容和目的，最后点击创建。
